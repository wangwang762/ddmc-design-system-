## 页面需求处理流程（必须严格遵守）

收到新的页面 UI 需求时，**按以下顺序执行，不得跳步**：

### 第一步：拿 UI Kit
- 先调用 `get_design_context` + `get_screenshot` 获取 Figma 设计稿
- 完整理解页面结构后再动手

### 第二步：优先复用现有组件
- 遍历已有组件库（见下方组件索引），有能直接用的**直接引用**
- 不得重复实现已有功能

### 第三步：扩展变体（次选）
- 如果现有组件**类型相近但缺少变体**，在原组件上新增 variant
- 必须同步更新组件的 `types.ts`、主组件文件、`index.ts`、`src/index.ts`、以及 App.tsx 演示
- **新增变体前需告知用户**："我将在 `<组件名>` 上新增 `<变体名>` 变体，是否可以？"

### 第四步：新增组件（最后手段）
- 确认现有组件库**完全无法覆盖**该需求时，才考虑新建组件
- **新增组件前必须征得用户同意**："当前需求无法用现有组件覆盖，我需要新建 `<组件名>` 组件，是否可以？"
- 获批后才可创建，文件结构遵循现有规范

## 组件索引

| 组件 | 适用场景 |

## 按钮 (Button)

**Figma node:** `8284:16216`

通用文字按钮，用于页面主流程操作。

### 尺寸选择

| 尺寸 | 高度 | 适用位置 |
|------|------|----------|
| `XLarge` | 42px | 页面底部固定操作栏（结算、确认支付、提交订单） |
| `Large` | 30px | 弹窗内主操作、卡片内操作 |
| `Medium` | 24px | 列表行内操作、标签旁的次要动作 |
| `Small` | 20px | 紧凑布局、标签式操作 |

### 类型选择

| 类型 | 视觉 | 适用场景 |
|------|------|----------|
| `Primary` | 绿色渐变填充 | 每个页面/弹窗只有一个主操作（去结算、确认、立即购买） |
| `Secondary` | 白色填充 + 绿色描边 | 次要操作、与 Primary 并列时的取消/返回 |

### 关键 Props

- `自适应={true}` (默认) → 按钮宽度由内容决定
- `自适应={false}` → 按钮撑满父容器，用于底部操作栏
- `disabled` → 条件未满足时（如购物车为空、表单未填写完整）

### 使用示例

```tsx
// 底部结算栏
<按钮 尺寸="XLarge" 类型="Primary" 自适应={false}>去结算</按钮>

// 弹窗操作对
<按钮 尺寸="Large" 类型="Secondary">取消</按钮>
<按钮 尺寸="Large" 类型="Primary">确认</按钮>

// 不可操作状态
<按钮 尺寸="XLarge" 类型="Primary" disabled>去结算</按钮>
```

## 购物车按钮 (Button/购物车)

**Figma node:** `4553:38967`

商品卡片右下角的圆形加购按钮，形状为 Super Squircle（iOS 风格圆角）。

### 尺寸选择

| 尺寸 | 适用卡片类型 |
|------|-------------|
| `26px` | 大尺寸商品卡片（首页推荐、搜索结果） |
| `22px` | 中尺寸商品卡片（分类列表） |
| `18px` | 小尺寸商品卡片（紧凑列表、横向滚动） |

### 状态选择

| 按钮状态 | 外观 | 使用时机 |
|----------|------|----------|
| `默认` | 绿色实心 squircle + 白色购物车图标 | 商品正常可购买 |
| `缺货加购` | 白色 squircle + 灰色描边 + 深色购物车图标 | 商品缺货但支持到货提醒/预购（用户可点击登记需求） |
| `不可选中` | 灰色实心 squircle + 白色购物车图标，按钮禁用 | 商品不可购买且无任何操作（超出配送范围、已下架） |

### Badge 规则

- 不传 `badge` 或 `badge={0}` → 不显示红点
- `badge={n}` n ≤ 99 → 显示数字
- `badge={n}` n > 99 → 显示 `99+`
- badge 表示该商品在购物车中的数量，有数量时才显示

### 使用示例

```tsx
// 正常商品卡片
<购物车按钮 尺寸="26px" 按钮状态="默认" badge={cartCount} onClick={handleAdd} />

// 缺货商品
<购物车按钮 尺寸="26px" 按钮状态="缺货加购" onClick={handleNotify} />

// 不可购买（禁用，不需要 onClick）
<购物车按钮 尺寸="26px" 按钮状态="不可选中" />
```

## 加减号 (Quantity Stepper)

**Figma node:** `14475:13875`

固定尺寸 69×22px。用于已加入购物车的商品数量调整，**不是**首次加购的入口（首次用购物车按钮）。

### 交互规则

| 场景 | Props 设置 |
|------|-----------|
| 正常调整数量 | `加号可点击={true}`（默认） |
| 达到库存上限 | `加号可点击={false}` → 加号变灰，减号仍可用 |
| 数量为 1 时点减号 | 由业务逻辑决定（从购物车删除 or 最低保留 1） |

### 使用示例

```tsx
const [qty, setQty] = useState(1)
const atStockLimit = qty >= product.stock

<加减号
  count={qty}
  onDecrement={() => setQty(q => Math.max(1, q - 1))}
  onIncrement={() => setQty(q => q + 1)}
  加号可点击={!atStockLimit}
/>
```

## 徽标 (Badge)

**Figma node:** `3787:17741`

叠加在 icon、头像、商品图片角落的提示角标，**绝对定位**于父元素上。

### 类型选择（核心决策）

| 类型 | 场景 | 示例 |
|------|------|------|
| `红点` | 有新消息/未读，但**不需要显示具体数量**时（低强度提示） | 消息入口有新通知、轻量提醒 |
| `数字` | 需要让用户明确知道**有几条/几件**时（中强度提示） | 购物车 3 件、消息 12 条 |
| `更多` | 数量超过 99，无需精确数字，只表示"很多"时 | count > 99 时自动展示 ··· |
| `左下直角` | 贴在卡片/图片**左上角**的文字状态标签（左下角直角贴合内容边缘） | "上新啦"、"爆款"、"预售" |

> 注意：`红点` vs `数字` 的选择标准是**用户是否需要知道具体数量**。
> 如果是"有没有"的问题 → 红点；如果是"有多少"的问题 → 数字。

### 尺寸选择（根据搭配的 icon 大小决定）

| 徽标尺寸 | 搭配的 icon 尺寸 | 典型场景 |
|----------|----------------|----------|
| `大` (16px) | 24px 及以上的 icon | 导航栏消息 icon、购物车 icon（26px）、头像（48px） |
| `小` (12px) | 18px 及以下的 icon | 列表行内小图标、紧凑卡片角标 |

### 描边规则

- `描边={false}`（默认）→ 叠加在**白色/浅色**背景上
- `描边={true}` → 叠加在**深色背景或彩色图片**上，1px 白色边框防止角标消失在背景中

### 使用示例

```tsx
// 消息入口 — 有新消息但不重要，用红点
<div style={{ position: 'relative' }}>
  <MessageIcon size={24} />
  <徽标 类型="红点" 尺寸="大" style={{ position: 'absolute', top: 0, right: 0 }} />
</div>

// 购物车入口 — 需要显示数量
<div style={{ position: 'relative' }}>
  <CartIcon size={24} />
  <徽标 类型="数字" 尺寸="大" count={cartCount} style={{ position: 'absolute', top: -4, right: -4 }} />
</div>

// 商品图片左上角状态标签
<div style={{ position: 'relative' }}>
  <img src={productImage} />
  <徽标 类型="左下直角" 尺寸="大" label="上新啦" style={{ position: 'absolute', top: 0, left: 0 }} />
</div>

// 深色背景上加描边
<徽标 类型="数字" 尺寸="大" count={5} 描边 />
```

## 标签 (Tag)

**Figma node:** `3617:16873`

行内文字标签，常与商品卡片、菜谱卡片搭配，标注属性、状态、来源等信息。

### 类型选择

| 类型 | 视觉 | 适用场景 |
|------|------|----------|

### 颜色语义

| 颜色 | 含义 | 典型用途 |
|------|------|----------|
| `绿色` | 积极/正常 | 新鲜、自营、有货 |
| `红色` | 紧迫/促销 | 促销、限时、缺货 |
| `蓝色` | 信息/特殊 | 预售、冷冻、进口 |
| `标准` | 中性 | 普通属性标注 |

### 尺寸选择

| 尺寸 | 高度 | 适用位置 |

| `28` | 28px | 详情页大标签、强调展示 |

| `18` | 18px | 紧凑卡片、标题行内标签 |

## 选择按钮 (Radio Button)

**Figma node:** `3460:14424`

单选控件，24×24px。用于互斥选项的选择（每次只能选一个）。

### 类型选择

| 类型 | 选中色 | 适用场景 |
|------|--------|----------|
| `默认` | #00B740 绿色 | 普通单选（地址、配送时段、规格选择） |
| `绿卡` | #EFD573 金色 | 会员专属选项（绿卡会员专享配送时段、绿卡专属规格） |

### 状态说明

| 状态 | 视觉 | 说明 |
|------|------|------|
| `未选中` | 白色/浅金圆圈 + 灰色描边 | 默认未选 |
| `选中` | 主色填充圆 + 白色勾选 | 已选中 |
| `不可选` | 灰色填充，按钮禁用 | 不可用选项（已售罄规格、不支持的配送时段） |

### 使用示例

```tsx
// 受控单选组
const [selected, setSelected] = useState<string>('morning')

{options.map(opt => (
  <选择按钮
    key={opt.id}
    状态={selected === opt.id ? '选中' : '未选中'}
    类型="默认"
    onClick={() => setSelected(opt.id)}
  />
))}

// 绿卡会员专属
<选择按钮 状态="选中" 类型="绿卡" onClick={handleSelect} />

// 不可用
<选择按钮 状态="不可选" 类型="默认" />
```

## 步进器 (Stepper)

**Figma node:** `10:56427`

数量输入控件。与 `<加减号>` 的区别：
- `<加减号>` → 购物车场景，固定尺寸 69×22px，有背景样式
- `<步进器>` → 通用数量输入，两种布局风格，用于订单填写、菜谱份数等

### 类型选择

| 类型 | 布局 | 适用场景 |
|------|------|----------|
| `大` | 自由浮动（minus gap box gap plus） | 订单数量、份量选择，有足够空间 |
| `小` | 分段边框（三格合一） | 紧凑列表行内、卡片内嵌数量 |

### 状态说明

| 状态 | 行为 |
|------|------|
| `默认` | 加减均可用 |
| `无法减少` | 减号禁用（数量已到最小值） |
| `无法增加` | 加号禁用（数量已到最大值/库存上限） |
| `无法修改` | 加减均禁用（只读展示） |

### 使用示例

```tsx
const [count, setCount] = useState(1)
const atMin = count <= 1
const atMax = count >= product.stock

<步进器
  类型="大"
  count={count}
  状态={atMin ? '无法减少' : atMax ? '无法增加' : '默认'}
  onDecrement={() => setCount(c => c - 1)}
  onIncrement={() => setCount(c => c + 1)}
/>

// 只读展示
<步进器 类型="小" 状态="无法修改" count={order.qty} />
```

## 弹窗 (Dialog)

**Figma node:** `3873:17915`

全屏遮罩 + 居中卡片（320px 宽，白色，r=12px）。通过 `ReactDOM.createPortal` 渲染到 `document.body`，自动锁定 body 滚动，支持 Escape 关闭。

### 三种结构对应 Figma 变体

| 结构 | 使用场景 | 关键 props |
|------|----------|-----------|
| 反馈类（无标题）| 纯文字告知，单个"知道了"按钮 | 只传 `description` |
| 确认类 | 需要用户确认/选择的操作 | `title` + `description`（可选）+ 2+ 个 `actions` |
| 输入类 | 收集用户单行文字输入 | `title` + `onInputChange` |

### actions 布局规则

| actions 数量 | stackActions | 渲染方式 |
|------------|--------------|---------|
| 1 个 | — | 全宽居中 |
| 2 个 | `false`（默认）| 左右平分 + 竖向分割线 |
| 2 个 | `true` | 垂直堆叠（文字过长时使用） |
| 3+ 个 | — | 垂直堆叠 |

### 按钮 kind 对应样式

| kind | 颜色 | 字重 | 使用场景 |
|------|------|------|---------|
| `primary`（默认）| 绿色 #00B740 | Semibold 600 | 主确认操作 |
| `secondary` | 黑色 #1A1A1A | Regular 400 | 取消、关闭 |
| `danger` | 红色 #FF3133 | Semibold 600 | 删除、清空等破坏性操作 |

### 使用示例

```tsx
// 反馈类 — 仅正文
<弹窗
  visible={open}
  description="您的订单已成功提交，预计30分钟内送达"
  actions={[{ label: '知道了', onClick: () => setOpen(false) }]}
  onMaskClick={() => setOpen(false)}
/>

// 确认类 — 双按钮
<弹窗
  visible={open}
  title="确认删除"
  description="删除后无法恢复，请谨慎操作"
  actions={[
    { label: '取消',   kind: 'secondary', onClick: () => setOpen(false) },
    { label: '确认删除', kind: 'danger',    onClick: handleDelete },
  ]}
  onMaskClick={() => setOpen(false)}
/>

// 输入类
<弹窗
  visible={open}
  title="修改收货地址备注"
  inputPlaceholder="请输入门牌号、楼层等"
  inputValue={note}
  onInputChange={setNote}
  actions={[
    { label: '取消', kind: 'secondary', onClick: () => setOpen(false) },
    { label: '保存', kind: 'primary',   onClick: handleSave },
  ]}
/>

// 带超长说明文本（可滚动）
<弹窗
  visible={open}
  title="用户协议"
  description={longText}
  scrollable
  actions={[{ label: '我已阅读并同意', onClick: handleAgree }]}
/>
```

## 开发规范

- 所有组件从 `@ddmc/design-system` 导入
- 不要用 HTML 原生 `<button>` 替代这些组件
- **颜色必须引用 token**：从 `@/tokens/colors` 导入 `dark / light / accentGreen / accentRed` 等，不得在组件内硬编码 hex 值
  - 例外：与色板无精确对应的 Figma 专属值（如 `#DCDCDC` 缺货描边、`#EFD573` 绿卡金色）在对应组件顶部定义为局部常量并注释说明
- Squircle 形状已内置，不要用 `border-radius` 模拟
