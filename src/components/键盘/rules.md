# 键盘 调用规则

> Figma node: `248:72583`

## 用这个组件的条件
在 mockup / 设计稿中，需要展示 iOS 系统键盘弹出状态时使用（如搜索页、输入地址页）。

## 不用这个组件的条件
- 真实 App 内无需手动添加，键盘由系统自动弹出
- 页面截图中无键盘交互时 → 不用此组件

## 变体选择

| 场景 | 模式 |
|------|------|
| 默认文字输入（首字母大写） | `字母大写` |
| 连续输入状态 | `字母小写` |
| 数字/特殊字符输入 | `符号` |

## 关键 Props

- `模式`：决定键盘当前显示的键位布局，默认为 `字母大写`
- 键盘底部已内置 Home Indicator，无需额外添加 `<HomeIndicator>`

## 示例

```tsx
// 搜索框聚焦，显示键盘
<键盘 模式="字母大写" />

// 已输入内容，切换小写
<键盘 模式="字母小写" />
```

## 常见误用

- 在键盘下方额外添加 `<HomeIndicator>`（键盘底部已内置，重复叠加）
- 用于真实输入交互（此组件是静态 mockup，不响应实际键入事件）

## 可替代组件

无。键盘仅用于 mockup 展示，无业务替代场景。
