# 加减号 调用规则

> Figma node: `14475:13875`

## 用这个组件的条件
购物车中已加购商品的数量调整，固定尺寸 69×22px。

## 不用这个组件的条件
- 首次加购入口 → 用 `购物车按钮`
- 独立于购物车的数量输入（订单填写、份量选择）→ 用 `步进器`

## 变体选择

无布局变体，只有状态差异：

| 场景 | 加号可点击 |
|------|-----------|
| 正常调整 | `true`（默认） |
| 已达库存上限 | `false`（加号变灰，减号仍可用） |

## 示例

```tsx
const [qty, setQty] = useState(1)
const atStockLimit = qty >= product.stock

<加减号
  count={qty}
  onDecrement={() => setQty(q => Math.max(1, q - 1))}
  onIncrement={() => setQty(q => q + 1)}
  加号可点击={!atStockLimit}
/>
```

## 常见误用

- 在购物车之外的独立场景使用（如订单份量选择），应用 `步进器`
- 把 `加减号` 和 `步进器` 交替使用造成视觉不一致
- 最小值减到 0 后不限制，导致出现负数

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 独立于购物车的数量输入 | `步进器` |
| 首次加购入口 | `购物车按钮` |
