# 购物车按钮 调用规则

> Figma node: `4553:38967`

## 用这个组件的条件
商品卡片右下角的加购入口，首次加购场景专用。

## 不用这个组件的条件
- 已在购物车中调整数量 → 用 `加减号`
- 页面级主操作 → 用 `按钮`

## 变体选择

**尺寸：**

| 卡片类型 | 尺寸 |
|---------|------|
| 大尺寸商品卡片（首页推荐、搜索结果） | `26px` |
| 中尺寸商品卡片（分类列表） | `22px` |
| 小尺寸商品卡片（紧凑列表、横向滚动） | `18px` |

**按钮状态：**

| 商品状态 | 按钮状态 |
|---------|---------|
| 正常可购买 | `默认`（绿色实心） |
| 缺货但可预购/到货提醒 | `缺货加购`（白色+灰描边） |
| 不可购买且无操作（下架、超出配送范围） | `不可选中`（灰色禁用） |

## 关键 Props

- `badge`：该商品在购物车中的数量，0 或不传 = 不显示，>99 自动显示 `99+`

## 示例

```tsx
// 正常商品
<购物车按钮 尺寸="26px" 按钮状态="默认" badge={cartCount} onClick={handleAdd} />

// 缺货
<购物车按钮 尺寸="26px" 按钮状态="缺货加购" onClick={handleNotify} />

// 不可购买
<购物车按钮 尺寸="26px" 按钮状态="不可选中" />
```

## 常见误用

- 已加购后继续展示购物车按钮而不切换为 `加减号`（加购后应切换状态）
- 把购物车按钮用在页面底部做主操作（应用 `按钮`）
- 尺寸与卡片尺寸不匹配（大卡片用 18px，视觉比例失调）

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 购物车内已加购数量调整 | `加减号` |
| 页面/弹窗主操作 | `按钮` |
