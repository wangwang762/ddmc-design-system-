# Cell 调用规则

> Figma node: Cell

## 用这个组件的条件
列表行，包含两类用途：
- `GroupedRow`：通用设置/信息列表行（地址列表、订单条目、设置项）
- `活动条`：购物车/结算页底部的满减活动提示条

## 不用这个组件的条件
- 需要展示商品图片+价格+加购 → 用 `商品卡片`
- 页面级主操作 → 用 `按钮`

## 变体选择

**GroupedRow 右侧类型：**

| 场景 | 类型 |
|------|------|
| 仅展示信息，可点击跳转 | `默认`（右侧文字+箭头） |
| 行内触发操作 | `按钮` |
| 开关设置项 | `开关` |
| 互斥单选项 | `单选按钮` |

**活动条尺寸：**

| 场景 | 尺寸 |
|------|------|
| 购物车页满减条 | `大` |
| 结算页紧凑满减条 | `小` |

**活动条左侧类型：**

| 左侧内容 | 类型 |
|---------|------|
| 文字标签（"满减"） | `标签` |
| 品牌/活动图片 | `图片` |

## 示例

```tsx
// 地址列表行
<GroupedRow
  主标题="家"
  副标题="上海市浦东新区XX路XX号"
  类型="默认"
  showArrow
  onClick={() => navigate('/address/edit')}
/>

// 设置开关行
<GroupedRow
  主标题="联系不上时允许放门口"
  类型="开关"
  开关Active={enabled}
  on开关Change={setEnabled}
/>

// 购物车满减条
<活动条
  尺寸="大"
  左侧类型="标签"
  标签文字="满减"
  活动文字="再买 ¥12.5 可减 ¥3"
  去凑单文字="去凑单"
  onCtaClick={handleGoFill}
/>
```

## 常见误用

- 用 Cell 展示商品（有图片+价格+加购的场景应用 `商品卡片`）
- 满减活动条用 `GroupedRow` 而不用 `活动条` 变体（样式和语义完全不同）
- 手写列表行而不用 `GroupedRow`，丢失统一的右侧交互模式

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 可购买商品展示 | `商品卡片` |
| 页面主操作 | `按钮` |
