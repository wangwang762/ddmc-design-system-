# 标题栏 调用规则

> Figma node: 标题栏

## 用这个组件的条件
二级及以下页面的顶部导航栏（商品详情、订单详情、设置页等）。

## 不用这个组件的条件
- 一级页面（有底部导航的页面）→ 通常不需要标题栏，或使用特殊定制头部
- 弹窗内部标题 → 用弹窗的 `title` prop

## 变体选择

**左侧操作（互斥）：**

| 场景 | 左操作 |
|------|--------|
| 普通返回上一页 | `返回`（默认） |
| 全屏弹层/模态页关闭 | `close` |
| 个人主页、用户中心顶部 | `用户信息` |
| 首页等无需返回的页面 | `无` |

**右侧（文案 与 图标入口不可同时存在）：**

| 右侧内容 | 使用场景 |
|---------|---------|
| `右侧文案` | 编辑、筛选等文字操作 |
| `show购物车` | 商品详情页、搜索页 |
| `show分享` | 商品详情页 |
| `show小程序` | 需要跳转小程序时，独占右侧 |

## 示例

```tsx
// 商品详情页
<标题栏
  标题="商品详情"
  左操作="返回"
  onBack={() => navigate(-1)}
  show购物车
  购物车Badge={cartCount}
  on购物车Click={() => navigate('/cart')}
  show分享
  on分享Click={handleShare}
/>

// 设置页
<标题栏 标题="账号设置" 左操作="返回" onBack={() => navigate(-1)} />

// 用户信息头部
<标题栏
  左操作="用户信息"
  头像Src={user.avatar}
  用户名={user.name}
  右侧文案="编辑"
  onRightTextClick={() => navigate('/profile/edit')}
/>
```

## 常见误用

- 一级页面（有底部导航）也加标题栏，造成双重导航
- 弹窗内部放标题栏（弹窗有自己的 title prop）
- 右侧同时传 `右侧文案` 和图标入口（两者不可共存）

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 首页顶部地址+消息入口 | `定位` |
| 弹窗标题 | `弹窗` 的 `title` prop |
