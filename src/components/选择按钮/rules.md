# 选择按钮 调用规则

> Figma node: `3460:14424`，24×24px

## 用这个组件的条件
互斥单选场景（每次只能选一个）：地址、配送时段、规格选择。

## 不用这个组件的条件
- 开关类设置项（通知、免打扰）→ 用 `开关`
- 数量选择 → 用 `步进器`

## 变体选择

| 场景 | 类型 |
|------|------|
| 普通单选 | `默认`（#00B740 绿色） |
| 会员专属选项（绿卡专享配送时段/规格） | `绿卡`（#EFD573 金色） |

**状态：**

| 状态 | 说明 |
|------|------|
| `未选中` | 默认未选 |
| `选中` | 已选中 |
| `不可选` | 已售罄规格、不支持的配送时段 |

## 示例

```tsx
const [selected, setSelected] = useState('morning')

{options.map(opt => (
  <选择按钮
    key={opt.id}
    状态={selected === opt.id ? '选中' : '未选中'}
    类型="默认"
    onClick={() => setSelected(opt.id)}
  />
))}

// 不可用
<选择按钮 状态="不可选" 类型="默认" />
```

## 常见误用

- 用选择按钮实现多选（此组件只有单选语义，多选用 checkbox）
- 把开关拿来做单选（语义错误：开关是 on/off，选择按钮是 N 选 1）
- 不可选中状态（如售罄规格）仍响应点击事件

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 开/关二态设置 | `开关` |
| 数量输入 | `步进器` |
| 多选场景 | 原生 checkbox（设计系统未封装） |
