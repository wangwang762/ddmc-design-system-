# 按钮 调用规则

> Figma node: `8284:16216`

## 用这个组件的条件
页面或弹窗中需要用户触发主流程操作时使用。

## 不用这个组件的条件
- 商品卡片加购 → 用 `购物车按钮`
- 数量增减 → 用 `加减号` 或 `步进器`
- 禁止用 HTML 原生 `<button>` 替代

## 变体选择

**尺寸：**

| 场景 | 尺寸 |
|------|------|
| 页面底部固定操作栏（结算、确认支付、提交订单） | `XLarge`（42px） |
| 弹窗内主操作、卡片内操作 | `Large`（30px） |
| 列表行内操作、标签旁次要动作 | `Medium`（24px） |
| 紧凑布局、标签式操作 | `Small`（20px） |

**类型：**

| 场景 | 类型 |
|------|------|
| 每个页面/弹窗的唯一主操作 | `Primary`（绿色渐变） |
| 与 Primary 并列的取消/返回 | `Secondary`（白色+绿描边） |

## 关键 Props

- `自适应={true}`（默认）→ 宽度由内容决定
- `自适应={false}` → 撑满父容器，底部操作栏必须设为 false
- `disabled` → 条件未满足时（购物车为空、表单未填完）

## 示例

```tsx
// 底部结算栏
<按钮 尺寸="XLarge" 类型="Primary" 自适应={false}>去结算</按钮>

// 弹窗双按钮
<按钮 尺寸="Large" 类型="Secondary">取消</按钮>
<按钮 尺寸="Large" 类型="Primary">确认</按钮>

// 禁用状态
<按钮 尺寸="XLarge" 类型="Primary" 自适应={false} disabled>去结算</按钮>
```

## 常见误用

- 商品卡片右下角加购用 `<按钮>`，应用 `购物车按钮`（尺寸和交互模式不匹配）
- 一个页面/弹窗出现多个 `类型="Primary"`，Primary 全局唯一，其余用 Secondary
- 卡片/列表行内用 XLarge，XLarge 仅用于底部固定操作栏
- 用 HTML 原生 `<button>` 替代，丢失 Figma 视觉规范和无障碍语义

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 商品卡片右下角加购 | `购物车按钮` |
| 购物车内数量调整 | `加减号` |
| 独立场景数量输入 | `步进器` |
