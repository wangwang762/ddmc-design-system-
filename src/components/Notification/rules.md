# Notification 使用规则

Figma node: 4131:63108

## 场景

页面顶部或模块内的单行提示条，不可换行，不打断用户操作。

## 变体决策

### 重要度（背景色）

| 重要度 | 背景 | 适用场景 |
|--------|------|---------|
| `默认` | 白色半透明 + blur | 叠在图片/渐变背景上 |
| `一般` | 浅灰 #F5F5F5 | 外部容器已是白色，需与背景区分 |
| `重要` | 浅红 #FFF5F5 | 警示、错误、重要通知 |

> 注意：`重要度=默认` + `左侧=标签` 时背景为纯白 `#FFFFFF`（不带 blur）。

### 左侧修饰

| 左侧 | 效果 | 使用时机 |
|------|------|---------|
| `默认` | 仅文案 | 普通提示 |
| `icon` | Icon(16px) + 文案 | 有明确语义图标辅助理解时 |
| `标签` | pill 标签 + 文案 | 需要强调提示类别（如"温馨提示"） |

标签颜色随重要度：
- `默认/一般`：深灰底 + 白字
- `重要`：浅红底 + 红字

### 可关闭

默认 `false`，用户主动关闭时传 `可关闭={true}` + `onClose`。

## 示例

```tsx
// 最简
<Notification 文案="您的订单正在配送中" />

// 重要 + 图标
<Notification 重要度="重要" 左侧="icon" iconName="Danger" 文案="支付失败，请重试" />

// 一般 + 标签（在白色容器内）
<Notification 重要度="一般" 左侧="标签" 标签文字="温馨提示" 文案="支持7天无理由退换货" />

// 可关闭
<Notification 文案="活动即将结束" 可关闭 onClose={() => setVisible(false)} />
```

## 常见误用

- 用 Notification 替代 Toast 显示操作反馈（Notification 是固定展示，Toast 是临时弹出）
- 在弹窗内部放 Notification（弹窗有自己的内容区域）
- 白色容器内用 `重要度="默认"`（blur 背景在白色上无效果，应用 `一般`）

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 轻量自动消失反馈 | `Toast` |
| 页面顶部系统/活动通知 | `NoticeBar` |
