# 步进器 调用规则

> Figma node: `10:56427`

## 用这个组件的条件
独立于购物车的通用数量输入（订单填写、菜谱份数、预约人数等）。

## 不用这个组件的条件
- 购物车内已加购商品数量调整 → 用 `加减号`

## 变体选择

| 场景 | 类型 |
|------|------|
| 空间充裕的数量选择（订单、份量） | `大`（minus gap box gap plus，自由浮动） |
| 紧凑列表行内、卡片内嵌 | `小`（三格合一分段边框） |

**状态：**

| 状态 | 行为 |
|------|------|
| `默认` | 加减均可用 |
| `无法减少` | 减号禁用（已到最小值） |
| `无法增加` | 加号禁用（已到最大值/库存上限） |
| `无法修改` | 加减均禁用（只读展示） |

## 示例

```tsx
const [count, setCount] = useState(1)

<步进器
  类型="大"
  count={count}
  状态={count <= 1 ? '无法减少' : count >= stock ? '无法增加' : '默认'}
  onDecrement={() => setCount(c => c - 1)}
  onIncrement={() => setCount(c => c + 1)}
/>

// 只读
<步进器 类型="小" 状态="无法修改" count={order.qty} />
```

## 常见误用

- 在购物车内用步进器，购物车只用 `加减号`（步进器没有购物车联动语义）
- 大型步进器嵌进紧凑列表行（用 `小` 类型或换 `加减号`）
- 不同页面混用大/小类型导致风格不统一

## 可替代组件

| 场景 | 替代组件 |
|------|---------|
| 购物车内数量调整 | `加减号` |
| 首次加购 | `购物车按钮` |
